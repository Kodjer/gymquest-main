# Автоматическая Генерация Квестов в GymQuest

## Обзор

GymQuest теперь использует систему **автоматической генерации квестов**, которая создает персонализированные задачи на основе профиля пользователя.

## Как это работает

### 1. Профиль пользователя
После регистрации пользователь заполняет анкету (OnboardingQuestionnaire), где указывает:
- Опыт тренировок (начинающий, средний, продвинутый)
- Предпочитаемые типы тренировок (силовые, кардио, йога, и т.д.)
- Цели фитнеса (похудение, набор массы, здоровье)
- Доступное время для тренировок
- Физические параметры (возраст, вес, рост)

### 2. Банк квестов
Приложение содержит более 50 шаблонов квестов в файле `src/lib/questBank.ts`, организованных по:
- **Категориям**: strength, cardio, flexibility, wellness
- **Уровням сложности**: easy, medium, hard
- **Базовым наградам XP**: от 30 до 250 XP

### 3. Умная генерация
Система автоматически:
- Подбирает квесты на основе предпочтений пользователя
- Определяет сложность в зависимости от уровня игрока и количества выполненных квестов
- Постепенно увеличивает сложность по мере прогресса
- Корректирует награды XP на основе уровня игрока

### 4. Прогрессивное усложнение
```typescript
// Логика определения сложности:
- Первые 5 квестов → легкие (easy)
- Уровень 1-3 → легкие (easy)
- Уровень 4-7 → средние (medium)
- Уровень 8+ → сложные (hard)

// Бонус XP за уровень:
XP = базовый_XP × (1 + (уровень - 1) × 0.1)
// +10% XP за каждый уровень
```

## API Endpoints

### POST /api/quests/generate
Генерирует 3 новых квеста для пользователя на основе его профиля.

**Ограничения:**
- Максимум 5 активных квестов одновременно
- Требуется заполненный профиль (onboarding)

**Ответ:**
```json
{
  "success": true,
  "quests": [...],
  "count": 3,
  "difficulty": "medium",
  "categories": ["strength", "cardio"]
}
```

### GET /api/quests
Получить все квесты пользователя

### PUT /api/quests/:id
Обновить статус квеста (завершить/отменить)

### DELETE /api/quests/:id
Удалить квест

## Новые поля в базе данных

### Модель Quest
```prisma
model Quest {
  id          String   @id @default(cuid())
  userId      String
  title       String
  description String?  @db.Text
  xpReward    Int
  difficulty  String   // easy, medium, hard
  category    String   // strength, cardio, flexibility, wellness
  status      String   @default("pending")
  isGenerated Boolean  @default(true)
  assignedAt  DateTime @default(now())
  completedAt DateTime?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

### Модель Player
```prisma
model Player {
  id                  String   @id @default(cuid())
  userId              String   @unique
  level               Int      @default(1)
  xp                  Int      @default(0)
  onboardingCompleted Boolean  @default(false)
  completedQuests     Int      @default(0)
  lastQuestGenerated  DateTime?
  createdAt           DateTime @default(now())
  updatedAt           DateTime @updatedAt
}
```

## Компоненты UI

### GenerateQuestsButton
Кнопка для генерации новых квестов с индикатором загрузки и обработкой ошибок.

### QuestCard
Улучшенная карточка квеста с:
- Эмодзи-иконками категорий
- Бейджами сложности и категории
- Раскрывающимся описанием
- Индикатором награды XP

## Категории квестов

| Категория | Эмодзи | Примеры |
|-----------|--------|---------|
| Strength | 💪 | Отжимания, приседания, планка |
